    Hur man begår förändringar utan staging.
  14. git log -p
    Detta kommando visar Commitys historia, inklusive alla filer och deras ändringar.
  16. git log --stat
    Denna kommando för att visa någon statistik om förändringarna i varje commit, inklusive rad (er) ändrade och filnamn.

    Denna kommando för att visa endast ostiga ändringar som standard på fil som är all_checks.py.
  20. git add -p
    Frågar om du vill ställa ändringar eller inte.
  22. git mv oldfile newfile

  24. git reset HEAD filename
    Hur man återställer stegvisa ändringar i Git.
  26. git commit --amend
    För att konfigurera usernamn i git.
  28. git revert comit_id_here
    Ger dig möjlighet att ändra och lägga till ändringar i det senaste engagemanget.
  30. git checkout branch_name
    För att gå till annat branch.
  32. git checkout -b branch_name
    För att skappa en ny branch.
  34. git merge branch_name
    Hur man slår samman två grenar i Git.
  36. git merge --abort
    Hur man avbryter en motstridig sammanslagning i Git.
  38. git remote -v
    Hur man ser URLs i Git.
  40. git push
    Så här driver du ändringar till en fjärrrepo i Git.
  42. git branch -r
    Hur man kontrollerar avlägsna grenar som Git spårar.
  44. git log origin/main
    Hur man kontrollerar den aktuella commit-loggen för en fjärrrepo i Git.
  46. git remote update
    Hur man hämtar innehållet i avlägsna grenar i Git utan att automatiskt slås samman.
  48. git push --delete origin branch_name_here
    För att ta bort en branch i git.
  50. git push -f
    Hur man tvingar fram en push-begäran i Git.
  52. git stash pop
    Denna kommando Återställer filer men raderar sedan den applicerade stashen.
  54. git rm --cached
    För att ta bort helt filens innehåll från indexet.
  56. git restore
    För att återställa den raderade filen.
  58. git tag
    För att lägga till tagg i git.
  60. git worktree list
    För att kunna se worktree i git.
