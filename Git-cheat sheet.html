Git-cheat sheet

  1. git config -l
    Listar alla variabler som går att konfigurera i git bash.
    Kan användas för att se vilka värden som är satta och vilka variabler det finns att ändra.
  2. git config --global user.name "username"
    För att konfigurera usernamn i git.
  3. git config --global user.email "email@domain.com"
    Ändrar e-postadressen i den globala konfigurationsfilen.
  4. git config --global credential.helper 'cache --timeout 300'"
    Kommando som minskar cachetiden till 5 minuter.
  5. git init
    Skapar ett tomt Git repository eller initierar ett existerande.
  6. git add filename_here"
    För att plus en fil här.
  7. git add .
    Lägger till alla ändrade filer till den aktiva branchen. Punkten kan bytas ut mot --all
  8. git add fil*
    för att lägga till alla filer vissa börja med "fil".
  9. git status
    Listar vilka filer som skiljer sig ifrån HEAD och den aktiva branchen.
  10. git commit"
    Denna kommando jobbar med commit.
  11. git commit -m "your commit message here"
    Commitar HEAD till den aktiva branchen med ett meddelande.
  12. git commit -a -m"your commit message here"
    Hur man begår förändringar utan staging.
  13. git log
    Visar loggen över commits.
  14. git log -p
    Detta kommando visar Commitys historia, inklusive alla filer och deras ändringar.
  15. git show commit-id
    Visar en specifik commit utefter ID:t på commitet du vill visa.
  16. git log --stat
    Denna kommando för att visa någon statistik om förändringarna i varje commit, inklusive rad (er) ändrade och filnamn.
  17. git diff
    Visar skillnader i commiten och trädstrukturen.
  18. git diff all_checks.py
    Denna kommando för att visa endast ostiga ändringar som standard på fil som är all_checks.py.
  19. git diff --staged
    Visar alla cachade ändringar. Staged är synonymt till cached.
  20. git add -p
    Frågar om du vill ställa ändringar eller inte.
  21. git rm filename
    Tar bort den specificerade filen.
  22. git mv oldfile newfile
    För att byta namnen av fil.
  23. git checkout filename
    Kastar bort ändringar till den specificerade filen innan git add.
    Använd -- innan filnamnet om filnamnet är identiskt med ett branchnamn.
  24. git reset HEAD filename
    Hur man återställer stegvisa ändringar i Git.
  25. git reset HEAD -p
    Låter dig välja vilka ändringar som skall återställas.
  26. git commit --amend
    För att konfigurera usernamn i git.
  27. git revert HEAD
    Återställer den senaste commiten.
  28. git revert comit_id_here
    Ger dig möjlighet att ändra och lägga till ändringar i det senaste engagemanget.
  29. git branch branch_name
    Skapar en ny branch med det givna namnet under den nuvarande branchen i trädet.
  30. git checkout branch_name
    För att gå till annat branch.
  31. git branch
    Visar alla branches och markerar den nuvarande branchen i trädet med grön färg och en *.
  32. git checkout -b branch_name
    För att skappa en ny branch.
  33. git branch -d branch_name
    Tar bort den valda branchen.
  34. git merge branch_name
    Hur man slår samman två grenar i Git.
  35. git log --graph --oneline --all
    Visar loggen för alla brancher som en graf på en rad.
  36. git merge --abort
    Hur man avbryter en motstridig sammanslagning i Git.
  37. git add remote https://repo_here
    Lägger till ett remote repository till mitt lokala repository.
  38. git remote -v
    Hur man ser URLs i Git.
  39. git remote show origin
    Visar information om den namngivna remoten (origin).
  40. git push
    Så här driver du ändringar till en fjärrrepo i Git.
  41. git pull
    Hämtar de senaste ändringarna från remote repository och mergear med ditt lokala repo.
  42. git branch -r
    Hur man kontrollerar avlägsna grenar som Git spårar.
  43. git fetch
    Hämtar de senaste ändringarna från remote repository men utför inte en merge med ditt lokala repo.
  44. git log origin/main
    Hur man kontrollerar den aktuella commit-loggen för en fjärrrepo i Git.
  45. git merge origin/main
    Mergear ditt lokala repo med remote repot.
  46. git remote update
    Hur man hämtar innehållet i avlägsna grenar i Git utan att automatiskt slås samman.
  47. git push -u origin branch_name
    Pushar den valda branchen till remote repot.
  48. git push --delete origin branch_name_here
    För att ta bort en branch i git.
  49. git rebase branch_name_here
    Flyttar commits till den angivna branchen. (Var väldigt försiktig om du använder det här kommandot)
  50. git push -f
    Hur man tvingar fram en push-begäran i Git.
  51. git stash -u
    Stash förvarar ändringar i ett dirty directory men låter dig fortsätta arbeta i ett oförändrat directory. -u flaggan inkluderar untracked och ignorerade filer.
  52. git stash pop
    Denna kommando Återställer filer men raderar sedan den applicerade stashen.
  53. git rm -r -f <file name>
    Tvingar borttagning av filer och mappar rekursivt. (Bör också användas med försiktighet)
  54. git rm --cached
    För att ta bort helt filens innehåll från indexet.
  55. git clean
    Tar bort otrackade filer.
  56. git restore
    För att återställa den raderade filen.
  57. git show
    Visar saker, flera olika taggar kan läggas till för att visa olika saker.
  58. git tag
    För att lägga till tagg i git.
  59. git submodule [--quiet] [--cached]
    En submodule är ett repository innuti ett annat repository. --quiet flaggar visar endast felmeddelanden och --cached flaggan tvingar användning av det som är sparat i index istället för HEAD.
  60. git worktree list
    För att kunna se worktree i git.
